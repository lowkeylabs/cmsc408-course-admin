General workflow

There are many approaches to database creation. For this class I recommend the following workflow:

  • Create database then create tables and then insert data all at one time
  • Store all commands in single SQL file
  • Paste commands to a command line in one chunk
  • Design file for reuse / rebuilding
  • SQL executes commands from top to bottom in file, like an interpreter.

SQL Creating tables

SQL Tables

Tables correspond to relations.

Tables are the fundamental object in a database.

Tables are created using the CREATE TABLE command. As you can imagine, there are plenty of options.

See the MySQL CREATE TABLE documentation

CREATE TABLE SYNTAX

CREATE TABLE <table name> (
    [columns and data types],
    [integrity constraints],
    [lots of other goodies!]
)

CREATE TABLE example

CREATE TABLE

CREATE TABLE instructor ( ID int AUTO_INCREMENT, full_name varchar(20) NOT NULL, dept_name varchar(30), salary decimal(10,2) CHECK (salary > 0), office char(5) NOT NULL UNIQUE, PRIMARY KEY (ID), FOREIGN KEY (dept_name) REFERENCES departments (department_name) ); indentation and capitalization doesn’t matter

Features

  • columns, data types
  • column constraints
    • AUTO_INCREMENT
    • NOT NULL
    • CHECK
    • UNIQUE
  • table constraints
    • PRIMARY KEY
    • FOREIGN KEY

Data Types

  • Numeric types
  • Date and time
  • String
  • Spatial (POINT, LINE, etc.)
  • JSON (structure)

ALTER and DROP tables

ALTER TABLE

ALTER TABLE tbl_name [alter_option [, alter_option] …] [partition_options]

 

  • Alter options:

    • ADD columns or constraints
    • ALTER columns or constraints
    • DROP columns or constraints

  • Partition options:

    • Related to storage organization

See the MySQL ALTER TABLE documentation

DROP TABLE

DROP [TEMPORARY] TABLE [IF EXISTS] tbl_name [, tbl_name] … [RESTRICT | CASCADE]

 

  • Can drop multiple tables with a single command.
  • IF EXISTS modifier will drop only if the table exists. This is handy for eliminating errors in code.
  • RESTRICT and CASCADE are useful when there are keys.
  • DDL tip: put all drops at top of file.

See the MySQL DROP TABLE documentation

SQL CREATE

SET foreign_key_checks = 0; DROP TABLE IF EXISTS city; DROP TABLE IF EXISTS state; DROP TABLE IF EXISTS country; SET foreign_key_checks = 1;

CREATE TABLE city ( city_id int, city_name varchar(30), city_home_state varchar(5), PRIMARY KEY (city_id) );

CREATE TABLE state ( state_abbr varchar(5), state_name varchar(30), state_home_country_id int, state_capital_city_id int, PRIMARY KEY (state_abbr) );

CREATE TABLE country ( country_id int not null, country_name varchar(50) not null, country_capital_city_id int not null, PRIMARY KEY (country_id) );

ALTER TABLE city ADD FOREIGN KEY (city_home_state) REFERENCES state ( state_abbr ); ALTER TABLE state ADD FOREIGN KEY (state_home_country_id) REFERENCES country (country_id); ALTER TABLE state ADD FOREIGN KEY (state_capital_city_id) REFERENCES city (city_id); ALTER TABLE country ADD FOREIGN KEY (country_capital_city_id) REFERENCES city (city_id);

Inserting data into tables

INSERT INTO syntax

INSERT INTO relation VALUES (V1, …, VN); # Inserts a new row with all the values matching the order of the columns. # BAD IN PRACTICE! # Example: INSERT INTO instructor VALUES (123, ‘ALBERTO CANO’, NULL, 1234.56, ‘E4251’);

INSERT INTO relation (Ci, …, Cj) VALUES (Vi, …, Vj); # Inserts a new row with the values matching any order of the columns specified, # any other column not listed is assumed to be NULL. # Examples:

INSERT INTO instructor (ID, name, salary, office) VALUES (123, ‘ALBERTO CANO’, 1234.56, ‘E4251’);

INSERT INTO instructor (name, salary, office) VALUES (‘ALBERTO CANO’, 9876.5, ‘W0465’); – ID AUTO GENERATED

 

See the MySQL INSERT INTO documentation

Features

  • Bulk INSERT Can be used to insert multiple records at once.
  • Lots of flexiblity in variable and value lists.
  • Some DBMS can load CSV files directly. Otherwise a CSV file is converted into SQL INSERT commands and then SQL is run.
  • Quarto strategy:
    1. Load CSV in python dataframe using Pandas, then
    2. Create table using Pandas.

UPDATE data in existing tables

UPDATE syntax

UPDATE [LOW_PRIORITY] [IGNORE] table_reference SET assignment_list [WHERE where_condition] [ORDER BY …] [LIMIT row_count]

value: {expr | DEFAULT}

assignment: col_name = value

assignment_list: assignment [, assignment] …

 

See the MySQL UPDATE docs

UPDATE examples

UPDATE instructor SET salary = 4321.00 WHERE ID = 123; # SAFE. Will update the salary based on the PRIMARY KEY.

UPDATE instructor SET salary = 4321.00 WHERE name = ‘ALBERTO CANO’; # DANGER! Will update the salary for ALL # instructors with the given name.

UPDATE instructor SET salary = 4321.00; # DANGER! Will update everyone’s salary!

UPDATE instructor SET salary = salary * 1.05; # DANGER! Will update everyone’s # salary +5% respectively.

Deleting data from tables

DELETE

Opposite of SELECT with many common arguments. DELETE [LOW_PRIORITY] [QUICK] [IGNORE] FROM tbl_name [[AS] tbl_alias] [PARTITION (partition_name [, partition_name] …)] [WHERE where_condition] [ORDER BY …] [LIMIT row_count]

Examples

DELETE FROM instructor WHERE last_name like ‘LEON%’;

DELETE FROM employee where id=101;

 

See the MySQL DELETE statement docs

TRUNCATE

Removes all rows in a table and resets metadata (counters, indexes, etc)

TRUNCATE TABLE instructor;

 

See the MySQL TRUNCATE docs

 

DROP TABLE

Removes table from the database.

DROP TABLE instuctor; DROP TABLE IF EXISTS country;

 

See the MySQL DROP TABLE docs )

DDL and DML summary

DDL

  • Data Definition Language
  • Used to create databases and tables
  • Used to define table, row and field constraints
  • Operations generally done atomically, in one big push.
  • Create a file (e.g., HW7-DDL.sql) and add all the commands.
  • Test commands one-by-one in VS code.
  • Test entire block to make sure it runs.
  • Commands: CREATE TABLE, ALTER TABLE, bulk INSERT, DROP TABLE

DML

  • Data Manipulation Language
  • Used to update individual records and fields
  • Generally connected to Web/HTML front end
  • Sometimes connected to API
  • API is constructed separate from UI
  • Data integrity paramount
  • Code designed to run each operation as needed ad-hoc.
  • Commands: INSERT INTO, UPDATE, DELETE
